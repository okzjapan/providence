# モデル設計指針

## 1. モデルセグメント分割

単一モデルではなく条件別にセグメント分割する。芝とダートは根本的に異なる競技であり、混合モデルは性能が劣化する。

### 1.1 初期セグメント（Phase 3）

| セグメント | 対象 | 特徴量の特徴 |
|-----------|------|-------------|
| `jra_turf_experienced` | JRA 芝・2 戦目以降 | フルスペック特徴量 |
| `jra_dirt_experienced` | JRA ダート・2 戦目以降 | フルスペック特徴量 |

### 1.2 拡張セグメント（Phase 5）

| セグメント | 対象 | 特徴量の特徴 |
|-----------|------|-------------|
| `jra_turf_debut` | JRA 芝・新馬戦 | 血統・調教中心（コールドスタート対応） |
| `jra_dirt_debut` | JRA ダート・新馬戦 | 同上 |
| `jra_obstacle` | 障害レース | 完全に別の予想ファクター |
| `jra_straight` | 新潟千直等の特殊コース | 別モデルまたはフラグ |

### 1.3 初期に除外するレース

- **新馬戦**: コールドスタート問題。Phase 5 で専用モデル対応
- **障害レース**: 平地とは予想ファクターが根本的に異なる
- **外国馬の出走レース**: 国内データなし

## 2. ラベル設計

### 2.1 ランキング学習用ラベル

着順を相対スコアに変換:

| 着順 | スコア |
|------|--------|
| 1 着 | 3 |
| 2 着 | 2 |
| 3 着 | 1 |
| 4 着以下 | 0 |

ランキング学習（LambdaRank）は同一レース内での相対評価を最適化するため、メンバーレベルが異なるレース間でも適切に学習できる。

### 2.2 補助モデル用ラベル（Phase 5 アンサンブル向け）

| モデル | ラベル | 目的 |
|--------|--------|------|
| 回帰モデル | 走破タイム（秒） | 絶対的なパフォーマンス予測 |
| 1 着分類モデル | 1 着 = 1, それ以外 = 0 | 勝ち馬の特定 |
| 3 着内分類モデル | 3 着以内 = 1, それ以外 = 0 | 複勝圏の特定 |

## 3. アルゴリズム

### 3.1 ベースライン: LightGBM LambdaRank

```python
params = {
    "objective": "lambdarank",
    "metric": "ndcg",
    "ndcg_eval_at": [1, 3, 5],
    "learning_rate": 0.05,
    "num_leaves": 31,
    "min_data_in_leaf": 20,
    "feature_fraction": 0.8,
    "bagging_fraction": 0.8,
    "bagging_freq": 5,
    "verbose": -1,
}
```

- `group` パラメータに各レースの出走頭数を指定
- NDCG@1 は 1 着予測精度、NDCG@3 は 3 着以内の予測精度

### 3.2 アンサンブル（Phase 5）

4 種モデルの重み付け統合:

1. **ランクモデル（LambdaMART）**: 着順の相対評価
2. **回帰モデル**: 走破タイム予測
3. **1 着分類モデル**: 勝つかどうかの二値分類
4. **3 着内分類モデル**: 複勝圏内かどうかの二値分類

重みは検証セットでの回収率を最大化するように Grid Search で決定。

## 4. バリデーション

### 4.1 時系列分割（必須）

**未来のデータで過去を予測してはならない**。

```
[-- 訓練データ (2年間) --][-- 検証 (1ヶ月) --][-- テスト (1ヶ月) --]
                           ← スライド →
```

- 訓練: 2 年間（約 7,000 レース）
- 検証: 1 ヶ月（Early Stopping 用）
- テスト: 1 ヶ月（最終評価用）
- ウィンドウをスライドさせて複数期間で検証

### 4.2 評価指標

| 指標 | 用途 |
|------|------|
| NDCG@1, NDCG@3, NDCG@5 | ランキング品質の定量評価 |
| 回収率シミュレーション | 実戦的な評価。最終的な判断基準 |
| 的中率（参考） | 直接の最適化対象にしない |

**注意**: NDCG が良くても回収率が伴わない場合がある。期待値ベースの回収率シミュレーションを最終的な判断基準にする。

### 4.3 Early Stopping

- 検証セットの NDCG を監視し、改善が止まったら学習を停止
- `early_stopping_rounds=50` 程度から開始

## 5. 特徴量選択

### 5.1 手順

1. **全特徴量でベースラインモデルを学習**
2. **Feature Importance (gain)** で上位特徴量を確認
3. **Forward Selection**: 1 つずつ特徴量を追加し、検証セットの NDCG 改善を確認
4. **200+ 次元から 50-80 次元に絞り込む** のが一般的
5. 最終的な特徴量セットを `metadata.json` に保存

### 5.2 Boruta（補助的）

- 全特徴量のシャドーコピーを作成し、シャドーより重要な特徴量のみを採用
- Forward Selection との併用で確度を高める

## 6. コールドスタート問題（新馬戦）

新馬戦では過去走データが一切存在しないため、専用の対処が必要。

### 6.1 使える特徴量

| カテゴリ | 影響度 | 特徴量 |
|---------|--------|--------|
| 血統 | ~50% | 種牡馬系統、母父系統、産駒成績集計 |
| 調教 | ~30% | JRDB 調教指数、追い切りタイム |
| 厩舎 | ~10% | 調教師の新馬勝率 |
| 騎手 | ~5% | 騎手実績 |
| その他 | ~5% | 馬体重、枠順 |

### 6.2 アプローチ

- **モデル分離**: 新馬戦は `jra_turf_debut` / `jra_dirt_debut` 専用セグメントで学習
- **ベイズ推定**: 血統情報を事前分布として確率を更新する手法が有効
- 新馬戦は AI にとっても難易度が高いが、人間の予想精度も低いためエッジが生まれやすい

## 7. Plackett-Luce 確率計算の競馬対応

既存の `probability/plackett_luce.py` を競馬の券種体系に対応させる。

### 7.1 競馬の券種（9 種）

単勝 / 複勝 / 枠連 / 馬連 / ワイド / 馬単 / 3 連複 / 3 連単 / WIN5

### 7.2 計算量の課題

- 18 頭の 3 連単: 4,896 通り（オートレース 8 頭の 336 通りの約 15 倍）
- 初期は全組み合わせ列挙で問題ない
- リアルタイム予測でボトルネックになる場合はスコア上位 N 頭に限定して近似

### 7.3 枠連の特殊性

枠連は枠番（1-8）の組み合わせで、同枠に複数頭が入る。Plackett-Luce から枠連確率を導出するには、同枠の馬のいずれかが入着する確率を合算する必要がある。
