# 次のレースを予測する

現在発売中で最も締め切りが近いレースを自動選択し、3フェーズで予測を行う。

- **フェーズ1**（締め切り10分前）: 試走タイムを取得し **勝率予測（WinProb）** を出す
- **フェーズ2**（締め切り3分前）: 最新オッズを取得し **購入推奨** を出す
- **フェーズ3**（即時）: 結果を要約して提示する

## 戦略プリセット

デフォルトは **安定型**。ユーザーが別のプリセット名やオプションを指定した場合はそちらを優先する。
全プリセットは 2026年3月1日〜15日（344レース）のバックテストで検証済み。

| プリセット名 | 券種 | 候補数 | Kelly | ROI | 的中率 | 特徴 |
|---|---|---|---|---|---|---|
| **安定型**（デフォルト） | 単勝, ワイド | 24 | 0.25 | +18.5% | 27.3% | 的中率と回収率のバランスが良い |
| 手堅型 | ワイド | 24 | 0.25 | +24.4% | 20.6% | ROI最高。ワイドのみで堅実に回収 |
| 積極型 | 単勝, ワイド, 2連単 | 24 | 0.25 | +24.3% | 24.1% | 2連単を加えてリターンを伸ばす |
| 全券種 | 全券種 | 12 | 0.25 | -62.8% | 2.0% | 従来デフォルト。参考用として残す |

対応する CLI オプション:

```
安定型:    --ticket-types win,wide --max-candidates 24
手堅型:    --ticket-types wide --max-candidates 24
積極型:    --ticket-types win,wide,exacta --max-candidates 24
全券種:    （オプション指定なし）
```

## 手順

### 1. 開催確認とタイミング判定

```bash
providence scrape today
```

出力テーブルから **Vote Close（締め切り時刻）が最も早い場のレース** を自動選択する。

- 開催がない → 「本日は開催がありません」と伝えて終了

次に現在時刻と締め切り時刻の差を計算する。

```bash
date "+%H:%M:%S"
```

| 締め切りまでの時間 | 動作 |
|---|---|
| 既に締め切り過ぎ | 「このレースは締め切り済みです」→ 終了 |
| **3分以内** | フェーズ1をスキップし、フェーズ2を即座に実行 |
| **3〜10分** | フェーズ1を即座に実行 → 締め切り3分前まで待機 → フェーズ2 |
| **10〜45分** | 締め切り10分前まで待機 → フェーズ1 → 締め切り3分前まで待機 → フェーズ2 |
| **45分超** | 「締め切りまで○○分あります。近づいてから再度実行してください」→ 終了 |

選択した場・レース番号・締め切り時刻・各フェーズの予定時刻をユーザーに明示する。例:

```
伊勢崎 R3 / 締切 11:51
  フェーズ1（試走タイム+勝率）: 11:41 開始予定（6分後）
  フェーズ2（オッズ+購入推奨）: 11:48 開始予定（13分後）
  現在 11:35 — フェーズ1開始まで待機します
```

### 2. フェーズ1: 試走タイム取得 → 勝率予測（締め切り10分前）

締め切り10分前まで待機が必要な場合:

```bash
sleep <締め切り10分前までの秒数>
```

出走表を取得し、試走タイムを確認する。

```bash
providence scrape day --date <YYYY-MM-DD> --track <場名>
```

試走タイムを反映した勝率予測を実行する（**--save なし**）。

```bash
providence predict \
  --date <YYYY-MM-DD> \
  --track <場名> \
  --race <番号> \
  --ticket-types win,wide \
  --max-candidates 24
```

**試走タイムの確認:**
- predict 出力に「試走タイムが未取得の車があります」警告が出た場合 → 「試走タイム未取得のため予測精度が低下する可能性があります」と警告を出すが、**続行する**（締め切り10分前にはほぼ出ているはずだが、万一の場合も止めない）
- 勝率テーブル（WinProb）をユーザーに要約して提示する

「購入推奨は締め切り3分前にオッズを取得してから出します」と伝える。

### 3. フェーズ2: オッズ取得 → 購入推奨（締め切り3分前）

締め切り3分前まで待機する。

```bash
sleep <締め切り3分前までの秒数>
```

最新オッズを取得する（試走タイムも再同期される）。

```bash
providence scrape odds --date <YYYY-MM-DD> --track <場名> --race <番号>
```

最新オッズを反映して predict を **--save 付きで** 実行する。

```bash
providence predict \
  --date <YYYY-MM-DD> \
  --track <場名> \
  --race <番号> \
  --ticket-types win,wide \
  --max-candidates 24 \
  --save
```

### 4. フェーズ3: 結果の提示

フェーズ2の出力を表示した後、以下を日本語で要約する。

- 選択した場・レース番号・締め切り時刻・使用プリセット名
- 各車の勝率（上位3車を強調）
- フェーズ1から勝率に変化があればその旨
- 推奨券の一覧（券種・組み合わせ・推奨金額）
- 合計投資額
- skip_reason が出た場合はその理由を日本語で説明

### エラー時

- 開催がない → 「本日は開催がありません」
- 締め切りまで45分超 → 「近づいてから再度実行してください」
- 締め切り過ぎ → 「このレースは締め切り済みです」
- `scrape` 失敗 → エラー内容を表示し、リトライを提案
- `predict` で skip → skip_reason を日本語で説明
