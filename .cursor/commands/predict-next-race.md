# 次のレースを予測する

現在発売中で最も締め切りが近いレースを自動選択し、3フェーズで予測を行う。

- **フェーズ1**（締め切り10分前）: 試走タイムを取得し **勝率予測（WinProb）** を出す
- **フェーズ2**（締め切り3分前）: 最新オッズを取得し **ランク付き購入推奨** を出す
- **フェーズ3**（即時）: 結果を要約して提示する

## ランク定義

各券種（単勝・ワイド）について、信頼度(Confidence)と最良EV(Expected Value)でランクを判定する。

| ランク | 信頼度 | 最良EV | 検証実績 | 推奨 |
|---|---|---|---|---|
| **S** | ≥0.95 | ≥0.3 | ROI 105-108% (検証済み黒字) | **投票推奨** |
| **A** | ≥0.90 | ≥0.2 | ROI 87-91% (黒字圏に近い) | 投票可 |
| **B** | ≥0.80 | ≥0.1 | ROI ~85% (やや不利) | 慎重に |
| **C** | ≥0.50 | ≥0.0 | ROI ~80% (不利) | 非推奨 |
| **D** | <0.50 or EV<0 | - | ROI低い | 回避 |
| **E** | - | 候補なし | - | 投票不可 |

ランク判定手順:
1. predict 出力の `Confidence:` 値を確認する
2. Strategy Recommendations テーブルの EV 列の最大値を確認する
3. 上の表に照らしてランクを決定する
4. skip_reason が出た場合: その理由によりランクを判定する
   - `low_confidence` → D ランク（信頼度不足）
   - `no_market_odds` → E ランク（オッズ未取得）
   - `no_positive_ev_candidates` → D ランク（正のEV候補なし）
   - `rounded_below_minimum` → C ランク（候補はあるが金額に達しない）

**全ランクで買い目を表示する。** skip_reason が出た場合も、フィルタを緩和して再実行し候補を表示する。

## 手順

### 1. 開催確認とタイミング判定

```bash
providence scrape today
```

出力テーブルから **Vote Close（締め切り時刻）が最も早い場のレース** を自動選択する。

- 開催がない → 「本日は開催がありません」と伝えて終了

次に現在時刻と締め切り時刻の差を計算する。

```bash
date "+%H:%M:%S"
```

| 締め切りまでの時間 | 動作 |
|---|---|
| 既に締め切り過ぎ | 「このレースは締め切り済みです」→ 終了 |
| **3分以内** | フェーズ1をスキップし、フェーズ2を即座に実行 |
| **3〜10分** | フェーズ1を即座に実行 → 締め切り3分前まで待機 → フェーズ2 |
| **10〜45分** | 締め切り10分前まで待機 → フェーズ1 → 締め切り3分前まで待機 → フェーズ2 |
| **45分超** | 「締め切りまで○○分あります。近づいてから再度実行してください」→ 終了 |

選択した場・レース番号・締め切り時刻・各フェーズの予定時刻をユーザーに明示する。例:

```
伊勢崎 R3 / 締切 11:51
  フェーズ1（試走タイム+勝率）: 11:41 開始予定（6分後）
  フェーズ2（オッズ+購入推奨）: 11:48 開始予定（13分後）
  現在 11:35 — フェーズ1開始まで待機します
```

### 2. フェーズ1: 試走タイム取得 → 勝率予測（締め切り10分前）

締め切り10分前まで待機が必要な場合:

```bash
sleep <締め切り10分前までの秒数>
```

出走表を取得し、試走タイムを確認する。

```bash
providence scrape day --date <YYYY-MM-DD> --track <場名>
```

フィルタなしで勝率予測を実行する（**--save なし**。フェーズ1は勝率確認のみ）。

```bash
providence predict \
  --date <YYYY-MM-DD> \
  --track <場名> \
  --race <番号>
```

- 勝率テーブル（WinProb）と Confidence 値をユーザーに要約して提示する
- 「購入推奨は締め切り3分前にオッズを取得してから出します」と伝える

### 3. フェーズ2: オッズ取得 → ランク付き購入推奨（締め切り3分前）

締め切り3分前まで待機する。

```bash
sleep <締め切り3分前までの秒数>
```

最新オッズを取得する。

```bash
providence scrape odds --date <YYYY-MM-DD> --track <場名> --race <番号>
```

**単勝とワイドのそれぞれで predict を実行する。**

まず単勝（S ランク設定）:

```bash
providence predict \
  --date <YYYY-MM-DD> \
  --track <場名> \
  --race <番号> \
  --ticket-types win \
  --min-confidence 0.95 \
  --min-expected-value 0.3 \
  --max-candidates 5 \
  --fractional-kelly 0.1 \
  --save
```

次にワイド（S ランク設定）:

```bash
providence predict \
  --date <YYYY-MM-DD> \
  --track <場名> \
  --race <番号> \
  --ticket-types wide \
  --min-confidence 0.8 \
  --min-expected-value 0.3 \
  --max-candidates 3 \
  --fractional-kelly 0.1 \
  --save
```

**skip_reason が出た場合（S ランク不適合）:** フィルタを緩和して再実行し、そのランクの買い目を表示する。

```bash
providence predict \
  --date <YYYY-MM-DD> \
  --track <場名> \
  --race <番号> \
  --ticket-types win,wide \
  --max-candidates 5 \
  --fractional-kelly 0.1
```

この結果の Confidence と EV 値からランクを判定する。

### 4. フェーズ3: ランク付き結果の提示

フェーズ2の全出力を元に、以下の形式で日本語要約する。

```
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
  伊勢崎 R3 / 2026-04-05 / 締切 11:51
  Confidence: 0.9732
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  勝率予測（上位3車）:
    1位: 3番 (32.1%)
    2位: 5番 (24.8%)
    3位: 1番 (18.2%)

  ── 単勝 ── ランク: S
    推奨: 3番 単勝 200円 (EV: 0.42, オッズ: 4.3倍)
    推奨: 5番 単勝 100円 (EV: 0.31, オッズ: 5.1倍)
    合計投資: 300円

  ── ワイド ── ランク: B
    参考: 3-5 ワイド 100円 (EV: 0.12, オッズ: 3.2倍)
    ※ Sランク基準(EV≥0.3)に達していないため非推奨

  判定: 単勝はS（投票推奨）、ワイドはB（慎重に）
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```

表示ルール:
- S ランク: 「投票推奨」と明記。買い目を太字で表示
- A ランク: 「投票可」と明記
- B ランク: 「慎重に」と明記。Sランク基準との差を説明
- C-D ランク: 「非推奨」と明記。理由を付記
- E ランク: 「投票不可」と明記

### エラー時

- 開催がない → 「本日は開催がありません」
- 締め切りまで45分超 → 「近づいてから再度実行してください」
- 締め切り過ぎ → 「このレースは締め切り済みです」
- `scrape` 失敗 → エラー内容を表示し、リトライを提案
- `predict` で skip → ランク判定に使用し、緩和版の買い目を表示
